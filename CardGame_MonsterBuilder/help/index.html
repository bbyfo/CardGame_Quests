üéØ Getting Started

Monster Builder is a card-first UI for creating monster cards for your board game.

Quick Start

  1. Click "New Monster" to create a new monster card
  2. Edit the name by clicking on it
  3. Drag cost and harm icons from the palette onto the card
  4. Click buttons to open editors for complex fields
  5. Changes auto-save after 1 second

üé¥ Monster Card Fields

Core Identity

  • Name: Monster display name
  • Polarity: Light or Shadow alignment
  • Type Tags: Core identity (must match polarity)
  • Aspect Tags: Secondary creature types

Combat Stats

  • Block Cost: Icons needed to encounter/block
  • To Vanquish: Icons needed to defeat
  • On Hit: Harm effects when monster attacks
  • Move Distance: How far monster can move
  • Attack Range: How far monster can attack

Behavior

  • Move Strategy: Movement pattern (chase, patrol, etc.)
  • Habitat: Biomes where monster appears
  • Special Abilities: Unique monster powers

üé® Icon Types

Cost Icons

  • Red: Physical/Martial
  • Green: Nature
  • Blue: Magic/Arcane
  • Yellow: Light/Holy
  • Purple: Shadow/Curse
  • Iron: Equipment/Tech
  • Gold: Wealth
  • Any: Wildcard

Harm Icons

  • Wound: Physical damage
  • Exhausted: Fatigue
  • Dazed: Confusion
  • Poisoned: Ongoing damage
  • Cursed: Negative status
  • Stunned: Can't act
  • Burned: Fire damage
  • Frozen: Cold/slow

üîß Tips & Tricks

  • Use auto-save: Changes save automatically after 1 second of inactivity
  • Export regularly: Download JSON backups of your monsters
  • TypeTags are polarity-restricted: Light monsters can only have Light tags
  • Use descriptive names: Make monsters easy to find in the list
  • Add designer notes: Document balance decisions and playtesting feedback

‚ùì Troubleshooting

Monster won't save

Check that:

  • Monster has a name
  • TypeTags match the Polarity
  • Numeric values are within valid ranges (0-20)

Server offline warning

The app will use localStorage as a fallback. Data is still saved locally but won't sync to the server.

Drag and drop not working

Make sure you're dragging icons onto the correct fields (cost icons on cost fields, harm icons on harm fields).
